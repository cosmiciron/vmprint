VMPrint Transmuters

Markdown → AST

Converts Markdown to DocumentInput JSON — the native input format for the VMPrint layout engine and CLI.

Browser demo limitations
  • Images are not fetched. Image URLs in your Markdown will be skipped silently. To include images, convert them to base64 data URIs (data:image/png;base64,…) and use those directly as the image source in your Markdown.
  • For best output, use the CLI. Download the generated JSON and run it through the VMPrint CLI (vmprint --input doc.json --output doc.pdf), which supports all fonts and full multilingual shaping. Pasting the JSON into the AST → PDF demo also works, but that example is limited to the 14 standard PDF fonts and does not include multilingual text support.
